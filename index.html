01

PIPELINE

전체 작업 흐름

  1. Clinical visit history 구축RID별 방문, 진단, 인구학 정보, APOE 등 정리
  2. MRI 후보 목록 생성T1과 rs-fMRI 스캔의 image/series metadata 연결
  3. QC 정보 병합공식 PASS/FAIL, motion, artifact, comment 반영
  4. 방문별 대표 스캔 선택사전에 정의한 우선순위로 중복 시퀀스 처리
  5. 두 modality 교집합 확정같은 방문의 T1–rs-fMRI 페어만 분석 대상으로 유지
  6. 다운로드 및 무결성 검사파일 수, 압축 해제, DICOM/NIfTI 구조 확인
  7. BIDS 정리와 전처리명명 규칙, 로그, QC 결과를 함께 보관
02

DATA SELECTION

1. Clinical visit history

목표는 각 RID에 대해 “언제, 어떤 phase에서, 어떤 visit code로 방문했고, 당시 진단과 주요 공변량이 무엇이었는가”를 한 행 단위로 정리하는 것입니다.

권장 키

  • RID
  • PHASE
  • VISCODE / VISCODE2
  • 방문일 또는 exam date

주요 변수

  • 진단 및 진단 변화
  • 나이, 성별, 교육연수
  • APOE genotype
  • 임상 점수와 연구 단계
VISITS 테이블

실제 longitudinal visit 자료가 아니라 visit-code lookup 성격인지 확인하고 사용합니다.

03

DATA SELECTION

2. MRI candidate scans

한 방문에 같은 modality의 스캔이 여러 개 존재할 수 있으므로, 먼저 후보를 모두 보존한 뒤 선택 규칙을 적용합니다.

항목확인 내용선택에 미치는 영향
SequenceMPRAGE, IR-SPGR, resting-state EPI 등사전에 허용한 protocol만 유지
Repeat동일 방문 반복 촬영 여부QC가 좋은 repeat를 우선
Field strength1.5T / 3T분석 설계에 따라 제한 또는 공변량 처리
MultibandMB / non-MB혼합 시 protocol 효과를 기록
Series metadataTR, TE, voxel size, volumes극단적 protocol 차이 탐지
04

DATA SELECTION

3. Quality control

DEFAULT DECISION RULE

공식 QC PASS → 코멘트 검토 → protocol 적합성 → 대표 이미지 선택

제외 우선

  • Overall QC = FAIL
  • 심한 motion 또는 ghosting
  • 불완전한 brain coverage
  • 잘못된 sequence 또는 orientation

우선 선택

  • 공식 QC PASS
  • motion이 더 작은 run
  • 완전한 volume 수
  • 분석 protocol과 일치
Search와 QC가 충돌할 때

일반적으로 최종 선택은 QC 테이블과 영상별 코멘트를 우선하되, 모든 예외를 selection log에 기록합니다.

PREPROCESSING

BIDS organization

dataset/
├── dataset_description.json
├── participants.tsv
├── sub-XXXX/
│   └── ses-M00/
│       ├── anat/
│       │   └── sub-XXXX_ses-M00_T1w.nii.gz
│       └── func/
│           ├── sub-XXXX_ses-M00_task-rest_bold.nii.gz
│           └── sub-XXXX_ses-M00_task-rest_bold.json
└── derivatives/

원본 데이터와 전처리 산출물은 분리합니다. BIDS 이름에는 직접식별정보를 넣지 않고, ADNI RID와 내부 subject ID의 매핑은 접근 제한 위치에 보관합니다.

08

PREPROCESSING

rs-fMRI preprocessing

ConvertDiscard volumesSlice timingRealignmentCoregistrationNormalizationNuisance regressionFilteringSmoothing

사용 도구와 설정에 따라 일부 단계의 순서는 달라질 수 있습니다. 가장 중요한 것은 software version, parameter, censoring 기준을 기록하는 것입니다.

필수 QC 지표

  • Framewise displacement
  • Head motion summary
  • Temporal coverage
  • Normalization overlay

분석 전 확인

  • TR과 volume 수
  • Motion censoring 비율
  • Global/WM/CSF 처리 방식
  • Smoothing kernel

T1 MRI preprocessing

ReorientationBias correctionSkull strippingSegmentationSurface reconstructionSpatial normalization

T1은 rs-fMRI의 해부학적 기준 영상이며, cortical thickness, volume, tissue mask 또는 registration reference로 사용됩니다.
